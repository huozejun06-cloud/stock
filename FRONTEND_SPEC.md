```

  → 跳过 Landing Page
  → 直接进入 Overview 看板页

```

```
│                                               │

│   ① "先看风险，再看机会。"                         │  ← 64px+ 白→灰渐变
│   ② "先避开错会，再看成套风险。"                    │  ← 渐入替换
│   ③ "不管你下一单，只陪你想清楚。"                   │  ← 渐入替换
│   最后固定底字：                                   │
│   "交易前先把机会和风险隔开一页，                    │  ← 24px 灰色细体
│    别让一时冲动替你做决定。"                        │  ← 最终固定文案
│                                               │
│           [ 开始智能决策 ➔ ]                    │  ← 毛玻璃发光按钮
│                                               │
│  ── 向下滚动 ──   ← 淡出提示                    │
└───────────────────────────────────────────────┘
```

**技术实现**：
- 背景: Three.js 实时渲染粒子系统（模拟流动K线/云海/科技光线），不是MP4视频
- Hero 文字: GSAP 多段流式飘出，每段文字 opacity 0→1 淡入 + y 30→0 上移，stagger 间隔 2s
  核心 GSAP 参数：gsap.fromTo(targets, {opacity:0, y:30}, {opacity:1, y:0, duration:1.5, ease:'power2.out'})
- 副标题段落在视频中逐句替换淡入，非一次性展示，最终固定为核心底字
- CTA 按钮: 带呼吸灯边缘发光效果的毛玻璃按钮 (backdrop-blur + box-shadow animation)
- GSAP ScrollTrigger: 向下滚动时文字优雅向上飘动淡出

## 九、AI 编码神圣执行协议（强制硬约束）

> **⚠️ 研发红线：AI 在生成任何 Phase D 的组件代码时，必须无条件死磕本说明书。任何将"毛玻璃、GSAP、ECharts、特定十六进制颜色"私自降级为"普通、默认、纯黑白、无动画"的行为，均视为严重开发交付事故。**

### 9.1 启动前置反问机制（Pre-flight Check）
AI 在收到开发某一个具体页面或组件的指令后，**禁止直接吐出代码**。必须先回复以下核对信息：
1. 🎯 本次开发涉及本说明书的哪几个章节？
2. 🎨 本次代码中，我使用了哪几个具体十六进制強調色（必须打出 `#ff6b35` 或 `#7c3aed` 等）？
3. 🎬 本次组件中，我使用了什么动画（CSS 还是 GSAP？参数是多少）？

### 9.2 严禁出现的"偷懒代码牛皮癣"
